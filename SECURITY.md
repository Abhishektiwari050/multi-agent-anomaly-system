# Security Policy

We take security vulnerabilities seriously and strive to maintain a safe codebase. Please report any potential issues responsibly.

## Supported Versions

Only the latest release on the `master` branch is supported with security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, **please do not open a public issue.** Instead, report the issue directly to the project maintainers:

*   Send a detailed email containing the vulnerability details, steps to reproduce, and any proof of concept.
*   We will acknowledge receipt of your report within 48 hours and work with you to patch the issue before making it public.

## Process

1.  **Submission**: Submit your report.
2.  **Triage**: Maintainers will verify and isolate the vulnerability.
3.  **Fix**: A security patch will be developed and verified against our automated test suites.
4.  **Release**: A patched version will be merged into the `master` branch.
5.  **Disclosure**: Public disclosure will be published with credits to the reporter.
